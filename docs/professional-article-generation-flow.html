Professional Article Generation Flow

This document explains how the professional ophthalmology article pipeline works: PDF text extraction, OpenAI chart extraction, Gemini article writing, expansion, chart rendering, and final article assembly.

Audience: professional Article writer: Gemini Chart extraction: OpenAI gpt-5.4 Chart renderer: Chart.js Target: 850-1000 words

Executive Summary

1. PDF becomes source text The API extracts text and sends the professional content path up to the AI layer.
2. OpenAI extracts chart data OpenAI reads the PDF text first and returns verified numeric datasets for charts.
3. Gemini writes the article Gemini receives source text plus chart context and writes the professional Polish article.
The key separation is intentional: OpenAI extracts structured chart data, Gemini writes the professional article, and Chart.js renders final chart images from verified numbers.

High-Level Flow

PDF upload

The user uploads one or more PDFs from the article creation screen.

API request

The frontend calls POST /api/ai/generate with target audience set to professional.

Text extraction

The API extracts PDF text and normalizes it before passing it into the AI provider layer.

Chart pre-extraction

Before Gemini writes, OpenAI extracts up to two clinically relevant chart datasets from the PDF text.

Chart verification

Extracted chart values are checked against source text to reduce hallucinated numeric output.

Gemini article generation

Gemini writes the full professional article using source text and chart context.

Expansion safety pass

If professional content is below 800 words, Gemini expands it to 850-1000 words using only source-supported information.

SEO and metadata normalization

SEO title, description, keywords, tags, category, and excerpt are checked and repaired where needed.

Cover image generation

Gemini image generation creates a professional cover image, with fallback search if needed.

Chart rendering

Chart.js renders PNG charts from OpenAI-extracted data, with axis bounds and significance colors.

Placeholder replacement

Chart placeholders or {{CHART:dataset:type}} tokens are replaced with uploaded chart images.

Final response

The API returns title, content, excerpt, SEO metadata, category, tags, and generated image URLs.

Component Roles

Component Responsibility Important Rules
PDF extraction Extracts readable text from uploaded PDF files. Professional generation uses a larger text slice than patient content.
OpenAI gpt-5.4 Extracts structured chart datasets before article writing. Every number must appear verbatim in source text. Whole-dataset SD must come from the whole-dataset table.
Gemini Writes the professional Polish article using source text and available chart context. 850-1000 words, table-traced numbers, subgroup completeness, non-significant comparisons, endpoint labels.
Gemini expansion Expands professional articles under 800 words. May only expand using source-supported facts. No invented studies, authors, p-values, or clinical claims.
Chart.js Renders final PNG charts from extracted data. Uses axis bounds, significance colors, and chart type selection from extraction output.

Professional Article Rules

Required Sections

  1. ## Streszczenie redakcyjne
  2. ## Metodyka i populacja
  3. ## Kluczowe wyniki
  4. ## Interpretacja kliniczna
  5. ## Ograniczenia
  6. ## Zrodlo

Writing Constraints

  • Use ultra-precise academic Polish.
  • Target 850-1000 words.
  • Use zero fluff and high data density.
  • Do not invent numbers, citations, or clinical claims.
  • Keep all claims grounded in the PDF.

Accuracy Requirements

  • Numeric trace rule: every SD, RMSAE, p-value, percentage, mean, or sample size must trace back to a table or figure.
  • Non-significant comparisons: when a significant result is reported, the article must also name non-significant comparisons.
  • Subgroup completeness: long eyes, short eyes, and IOL type subgroups must include primary and secondary endpoint findings.
  • Endpoint labeling: primary endpoint findings and secondary endpoint findings must be labeled on first mention.
  • Zeroed-mean analysis: if mean PE adjusted to zero is reported, it must appear under Kluczowe wyniki, not as a limitation.

Chart Type Selection

Data Shape Chart Type Reason
Distribution statistics: min, Q1, median, Q3, max boxplot Shows distribution and spread.
Many categories, especially formula comparisons horizontalBar Improves label readability.
Cumulative thresholds or sequential intervals line Thresholds are ordered. Radar is not allowed for this.
Independent non-ordered dimensions radar Only valid when axes are truly independent dimensions.
Single percentage composition pie or doughnut Shows proportional parts.
Cumulative threshold data such as % eyes within +/-0.25D, +/-0.50D, +/-0.75D, +/-1.0D must be a line chart, not radar.

Whole-Dataset SD Tracing Rule

For whole-dataset SD comparisons, especially Hoffer QST, the extractor may only use the table explicitly labeled as the whole-dataset, overall, or primary results table. It must not pull values from short-eye, long-eye, IOL-type, subgroup, or supplemental tables.

Chart Rendering Rules

Significance Color Mapping

Status Color Meaning
best Green Best performer and statistically significant.
sig_worse Blue Significantly worse than the best performer.
ns Gray No statistically significant difference versus best.

Axis Scaling

  • Default: axis_min = min - 15% of range, axis_max = max + 15% of range.
  • Threshold line charts: axis_min = minimum value across all series - 5.
  • Threshold line charts: axis_max = 100.
  • This prevents small differences from being hidden by an unnecessary zero baseline.

Your task is to create a detailed professional review for ophthalmologists and optometrists.

CRITICAL: You must return a complete JSON object with fields:

Writing Style for the "content" field:
1. Zero Fluff: eliminate introductory phrases, transitional sentences, and meta-commentary.
2. Maximum Density: prioritize data, p-values, anatomy, protocols, endpoints, and exact results.
3. Length: aim for 850-1000 words when the source document contains enough substance.
4. Language: write in ultra-precise academic Polish.

Required content structure:
## Streszczenie redakcyjne
## Metodyka i populacja
## Kluczowe wyniki
## Interpretacja kliniczna
## Ograniczenia

Ground all claims strictly in the provided document. Do not hallucinate data.

NUMERIC ACCURACY - MANDATORY:
Before writing any numeric value (SD, RMSAE, p-value, percentage, mean, n), trace it to a specific table or figure.
If you cannot confirm a number exists verbatim, omit it entirely. Never report a value for one formula that belongs to another formula.

NON-SIGNIFICANT COMPARISONS - MANDATORY:
For every statistically significant finding you report, explicitly state which comparisons did NOT reach significance (p>=0.05).

SUBGROUP REPORTING - MANDATORY:
For each subgroup (oczy dlugie, oczy krotkie, typ IOL), report both the primary endpoint winner and notable secondary endpoint findings.

ENDPOINT LABELING - MANDATORY:
Label whether a finding comes from a primary endpoint (SD, RMSAE) or secondary endpoint (interval percentages, median AE).

ZEROED-MEAN ANALYSIS:
If the paper reports analysis after adjusting mean PE to zero, include it under:

This is a primary analytical result, not a limitation.

OpenAI Chart Extraction Prompt

Analyze the medical/scientific document and extract EXACTLY 2 clinically relevant numerical datasets for visualization.

ABSOLUTE ANTI-HALLUCINATION RULES:
- Every number must appear verbatim in the document.
- Do not round, estimate, interpolate, or extrapolate.
- Do not combine numbers from different tables/contexts.

HOFFER QST / WHOLE-DATASET SD NUMERIC TRACING RULE:
- When extracting SD values for whole-dataset comparison, use ONLY Table 2 or the explicitly labeled whole-dataset / primary results table.
- Do not use subgroup tables, long-eye tables, short-eye tables, IOL-type tables, or supplemental tables.
- If the same formula appears in multiple tables with different values, use the whole-dataset / primary results table.

MANDATORY CHART VARIETY RULE:
- When extracting 2 charts, use 2 visually different chart families.
- Bar-family means: bar, horizontalBar, stackedBar.
- If one chart is bar-family, the other must be non-bar.

CHART TYPE SELECTION RULE:
- Cumulative thresholds or sequential intervals must use "line", not "radar".
- Example: % eyes within +/-0.25D, +/-0.50D, +/-0.75D, +/-1.0D.
- Radar is only for truly independent, non-ordered dimensions.

AXIS SCALING:
- For every chart return axis_min and axis_max.
- Default: axis_min = min - 15% of range; axis_max = max + 15% of range.
- For cumulative interval line charts: axis_min = minimum value across all series - 5; axis_max = 100.

SIGNIFICANCE ENCODING:
- For bar, horizontalBar, stackedBar, return a significance array parallel to the first dataset.
- Values: "best", "sig_worse", "ns".
- If pairwise comparisons are not reported, set all values to "ns" and significance_source to "not_reported".
- For non-bar charts, significance = [] and significance_source = "not_reported".
